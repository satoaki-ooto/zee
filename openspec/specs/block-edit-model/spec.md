## ADDED Requirements

### Requirement: 不連続な複数領域を1操作として表現できる

編集モデル（diff / revision / reconcile）は、行ごとに分かれた **不連続な複数の文字範囲** の変更を、まとめて1つの編集操作として表現・適用・巻き戻しできなければならない（MUST）。単一連続領域を内包するバウンディング領域での近似によって、矩形に含まれない文字（`[0, left)` や `[right, 行末]`、選択行範囲外の行）を巻き込んで変更・消失させてはならない（MUST NOT）。

#### Scenario: 複数行の不連続範囲を1操作で適用

- **WHEN** 行 r0..=r1 のそれぞれで visual column `[left, right)` に当たる文字を取り除く編集を1操作として適用する
- **THEN** 各行で `[0, left)` と `[right, 行末]` だけが連結され、矩形の外側の文字は一切変更されない

#### Scenario: バウンディング近似で範囲外を巻き込まない

- **WHEN** 不連続な複数範囲の編集を適用する
- **THEN** それらを内包する単一連続領域では近似されず、範囲外（矩形に含まれない文字）は変更・消失しない

### Requirement: 1操作 = 1リビジョン（undo の原子性）

不連続な複数領域の1編集操作は、何行にまたがろうと **1回の undo でその操作全体がちょうど元に戻り**、**1回の redo で全体が再適用される**ものでなければならない（MUST）。操作の途中状態（N行処理した時点）が undo 履歴に独立したリビジョンとして現れてはならず、1操作に対して undo を複数回押さないと戻らない状態になってはならない（MUST NOT）。

#### Scenario: 1回の undo で全行が戻る

- **WHEN** 複数行にまたがる不連続領域の編集を1回行った直後に undo を1回する
- **THEN** 全行の変更が同時に巻き戻り、バッファは編集直前と完全一致する

#### Scenario: 1回の redo で全行が再適用される

- **WHEN** 上記 undo の直後に redo を1回する
- **THEN** 全行の変更が同時に再適用され、バッファは編集直後の状態に戻る

#### Scenario: 部分 undo が起きない

- **WHEN** 複数行の不連続編集を1回行った直後に undo を1回する
- **THEN** 一部の行だけが戻って他の行が残る状態にはならない

### Requirement: 既存の単一連続編集との後方互換

通常の1文字入力・連続範囲の選択/削除/コピーは、このモデル変更後も **観測可能な振る舞いが一切変わってはならない**（MUST NOT change）。通常編集の undo 粒度・カーソル追従・選択正規化は不変でなければならない（MUST）。

#### Scenario: 通常の1文字編集の undo 粒度が不変

- **WHEN** モデル変更後に通常の1文字入力・連続削除を行い undo する
- **THEN** 変更前と同一の粒度・カーソル位置で巻き戻る

#### Scenario: 既存 selection テストが回帰しない

- **WHEN** モデル変更後に既存の selection / delete_selection の振る舞いを検証する
- **THEN** 変更前と同一の結果になる
