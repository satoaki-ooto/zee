# Roadmap: zee — 矩形（ブロック）選択・編集モード

## 向かう先

zee に矩形（ブロック）選択モードを入れ、Emacs の rectangle 相当を扱えるようにする。
すなわち矩形での **選択・コピー・カット・ペースト・矩形挿入** のすべてを、通常の編集と同じ操作感（1操作＝1 undo）で行える状態を最終地点とする。

## 現在の区間: 矩形ペースト（貼り戻し）まで（土台の上に読み戻しを乗せる区間）

向かう先のうち、前区間で作った「破壊的編集（カット/削除）の土台」の上に、**矩形で切ったものを矩形として貼り戻す**ところまでを取りに行く。
この区間の重心は、clipboard を「矩形で切った」と区別できるようにすること（メタ情報の保持）と、貼り戻しを既存の compound diff 機構に乗せて 1操作＝1 undo に保つことにある。矩形挿入（string-rectangle）は、ペーストの土台ができてから次の区間で乗せる。

### この区間でやること

- clipboard が「矩形で切った」というメタ情報を保持できる（通常の kill と矩形の kill を区別できる）
- 矩形コピー/カットしたものを矩形として貼り戻せる（yank-rectangle：各行を1行ずつ下にずらして挿入）
- 矩形ペースト全体が **1回の undo で戻る**（前区間の compound diff 機構に乗せる）
- 挿入列が行末より右にある短い行を、スペースで padding してから貼り戻す
- 矩形ペースト用のキーバインドを、確定済みの `Ctrl-x r` プレフィックス方式に沿って割り当てる

### この区間でやらないこと

- 矩形挿入（string-rectangle：同一列に同じ文字列を一括挿入） — 次の区間。ペーストの土台ができてから乗せる
- 挿入プレビュー描画（貼り戻し前に結果を視覚確認する表示） — そもそも作らない。選択ハイライトで足りる
- 複数カーソルと矩形ペーストの併用 — 現状は単一カーソル運用であり、今は作らない
- tab / CJK 幅をまたぐ列計算の網羅的なエッジケース対応 — 基本の幅整合（grapheme width 準拠の列マッピング）まではこの区間に含む。変則ケースの作り込みは後回し（前区間からの継続）
- 各能力の「どう動作すべきか」「どう作るか」の詳細設計 — behavior-definition → requirement-definition → openspec の領域。このロードマップは方向と範囲までで止める

### 状態

- **着手前** ─ 2026-06-13 区間を切った。コードベース現状調査（`docs.local/rectangle-mode-ph2.md`）で、clipboard が `String` のみ・矩形メタ情報を持てないこと、矩形ペーストの entry point が未実装であることを確認済み。次は behavior-definition で「どう動作すべきか」を詰める。

## これまでの区間（要約）

- **矩形のカット/削除まで**（完了）: 編集・undo モデルを不連続複数領域対応に拡張。矩形選択→ハイライト→コピー→カット/削除まで実装。compound diff 機構（D1）と `Ctrl-x r` プレフィックスキーバインド（D6）を確定。

## 次の区間の候補

- 矩形挿入（string-rectangle：同一列に同じ文字列を一括挿入）
- tab / CJK 幅をまたぐ列計算の網羅的なエッジケース対応
- （必要になれば）複数カーソルと矩形選択の統合

## 残された曖昧さ

- clipboard の矩形メタ情報を「どこに・どう持つか」が未決（trait 拡張 / 内部の矩形 kill ring / 文字列へのマーカー埋め込みなど）。とくに SystemClipboard 経由でも矩形性を保てるかが論点 → 方向としては「矩形ペーストは通常 paste と別コマンドにする」までは確定。具体手段は openspec へ。
- 矩形ペースト後のカーソル最終位置の仕様が未確定（Emacs では矩形下端の次行・左端列）→ behavior-definition で詰める。
- 短い行の padding を「やる」に含めたが、padding の具体挙動（tab を含む行・行末ちょうどの扱い）は未定義 → behavior-definition で詰める。
- ~~矩形カット/削除の undo を「1操作＝1リビジョン」で実現する具体手段が未決~~ → **解決**: `CompoundDiff`（`OpaqueDiff` の順序付き列）を採用。バウンディング近似は C-2 違反で却下。
- ~~矩形モードのキーバインド具体割り当てが未確定~~ → **解決**: `Ctrl-x r` プレフィックス方式（`Ctrl-x r Space` で進入、`Ctrl-x r M-w` でコピー、`Ctrl-x r C-w` でカット、`Ctrl-x r d` で削除）。
- ~~既存 `reconcile()` の次元混在疑い~~ → compound 対応で `reconcile_compound()` を追加。単一 `reconcile()` は不変。次元混在は現状踏まず（単一カーソル運用で影響限定的）。
