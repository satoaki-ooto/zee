## 振る舞い定義: 矩形コピー

### 何のための対象か

ROADMAP「矩形のカット/削除まで」区間の読み取り系。
矩形選択範囲を、行ごとに切り出して連結し clipboard に入れる。バッファは変更しない。横断制約 C-4（visual column）・C-5（短い行は Emacs 準拠＝空扱い）を前提。

### 期待する振る舞い

濃度は Given/When/Then（切り出し規則がブレると cut と整合しなくなるため厳密に）。

- **基本（GWT）**
  - Given: 矩形が行範囲 `r0..=r1`、visual column 範囲 `[left, right)` で選択されている
  - When: 矩形コピー操作をする
  - Then: 各行 r（r0 から r1 へ昇順）について `[left, right)` に重なる部分文字列を切り出し、**行の出現順に改行 `\n` で連結**した文字列を clipboard に set する。バッファ・undo 履歴は一切変化しない。

- **短い行（C-5: Emacs 準拠＝空扱い、GWT）**
  - 行末 visual column < `left` の行 → その行の切り出しは **空文字列**（空白で埋めない）。連結時はその行分として空行が1つ入る（＝改行は維持される）。
  - 行末が `[left, right)` の途中までの行 → `[left, 行末]` までの存在する文字だけを切り出す（右側を空白で埋めない）。

- **コピー後（散文）**
  - コピー後は矩形選択を解除し通常編集状態に戻る（既存 `copy_selection_to_clipboard` が copy 後に `clear_selection()` する挙動に倣う）。
  - clipboard には**フラットな文字列**（行を `\n` で連結したもの）が入る。「矩形で切った」というメタ情報は持たせない（ROADMAP やらないこと、次区間）。

### 許さない振る舞い・守る制約

- **絶対にこうならない**:
  - コピーがバッファを1文字でも変更する。undo 履歴にリビジョンを作る（コピーは非破壊。diff は empty）。
  - ハイライト表示されている範囲と、実際に clipboard に入る範囲がずれる（C-4）。
  - 短い行で、選択していない仮想空間を空白として clipboard に混ぜる（C-5 の空扱いに反する。＝今は padding しない）。
  - clipboard の改行構造が、行ごと連結と食い違う（行数分の改行が保たれること）。
- **常に満たす前提**:
  - 切り出しの列計算は C-4 と同一尺度。highlight・cut と完全に同じ切り出し規則を共有する（3者で規則が分岐しない）。
  - C-5 の空扱い ↔ 空白パディングの切り替えは、**copy 内に判断を埋め込まず、共有の分岐点を通す**（後から padding に差し替え可能に）。

### 濃度

厳密(GWT)。理由: 切り出し規則が cut・highlight と共有される中核。ここがブレると3能力が不整合になる。

### 確定事項（ユーザ確認済み）

- **幅0矩形のコピー**: **no-op**（clipboard を触らない）。padding しない（_constraints C-5）。
- **末尾改行**: 行間のみに `\n` を入れ、最終行（r1）の末尾には改行を付けない（Emacs `copy-rectangle-as-kill` 準拠）。

### 残された曖昧さ（振る舞いレベル）

- なし。

### design へ送る実装論点（振る舞いではない）

- system clipboard（crossclip）と local clipboard で改行コード（`\n` / `\r\n`）の扱いが分かれないかは実装時に確認。

### 次にどうする

- selection / highlight / cut と合わせて requirement-definition へ。
