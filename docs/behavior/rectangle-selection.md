## 振る舞い定義: 矩形選択モード（進入と2次元範囲選択）

### 何のための対象か

ROADMAP「矩形のカット/削除まで」区間の、矩形編集すべての入口。
矩形モードに入り、(行, 列) の2次元範囲をアンカーと対角で選べる状態を作る。横断制約 `_constraints.md`（特に C-4 列は visual column）を前提とする。

### 期待する振る舞い

濃度は Given/When/Then（エディタの編集系であり、解釈ブレが後段の copy/cut に波及するため厳密に固定する）。

- **進入（Given/When/Then）**
  - Given: 通常編集状態で、カーソルがある grapheme 上にある
  - When: 矩形選択モードに入る操作をする
  - Then: その時点のカーソル位置が矩形の **アンカー角**（行・visual column の両方）として固定される。進入直後の矩形は「アンカー1点のみ」＝幅0・高さ1。

- **対角の決定（Given/When/Then）**
  - Given: 矩形モードに入っている（アンカー固定済み）
  - When: 通常の移動キーでカーソルを上下左右に動かす
  - Then: カーソル位置が **対角の角** となり、アンカーとカーソルが作る (行範囲 × visual column 範囲) が現在の矩形になる。アンカーは動かない。
  - 行範囲: アンカー行とカーソル行の min..max（両端含む）。
  - 列範囲: アンカー列とカーソル列の min..max。左右どちら向きに選んでも正規化され、向きに依存しない（既存 `selection()` の正規化思想を2次元に拡張）。

- **EOL を越える列（散文）**
  - 対角カーソルが、ある行では行末より右の visual column に来ることを許す。矩形の右端列は「カーソルの visual column」で決まり、各行の実際の文字長には縛られない。短い行で右端列に文字が無い場合の切り出し/削除の扱いは copy / cut 各ファイル（C-5: Emacs 準拠＝空扱い）。

- **キャンセル（Given/When/Then）**
  - Given: 矩形モードに入っている
  - When: キャンセル操作（`C-g` 相当）をする
  - Then: 矩形選択は破棄され、カーソル位置はそのまま、通常編集状態に戻る。バッファは変化しない。

- **進入の方式（散文・ROADMAP 必須項目）**
  - zi は Shift 修飾を持たない（rectangle-mode.md / README TODO）。よって「Shift+移動で矩形を広げる」方式は採れない。
  - 既存 `select-all` と同様の **多段プレフィックス方式**で矩形モードに入る、を方式の第一候補とする。進入後は通常の移動キーで対角を動かす（モーダル）。

### 許さない振る舞い・守る制約

- **絶対にこうならない**:
  - 進入後にアンカーが移動する（アンカーは進入時点で固定。動くのは対角のみ）。
  - 選択の向き（上→下／右→左 など）によって、確定する矩形領域が変わる。正規化されない。
  - 矩形モードで列を char index として測り、CJK / tab を含む行で見た目とずれる（→ C-4 違反）。
  - 矩形モード進入が、通常の連続選択（`begin_selection`）の振る舞いを書き換える（→ C-3 後方互換違反）。通常選択と矩形選択は両立せず、どちらか一方が active。
- **常に満たす前提**:
  - 矩形モード中はバッファ（Rope）を変更しない。選択は読み取り専用の状態（rectangle-mode.md の Phase 1 stateless と同じ位相）。
  - 現状は単一カーソル運用。複数カーソルとの併用は作らない（ROADMAP やらないこと）。

### 濃度

厳密(GWT) を主、進入方式と EOL 越えは散文。理由: 進入・対角・正規化は後段 copy/cut の前提になるため固定する。進入方式の具体キーは未確定なので散文に留める。

### 確定事項（ユーザ確認済み）

- **幅0矩形（列範囲が空、行のみ）**: 選択状態にはなれる（矩形挿入＝次区間の布石）が、copy/cut は **no-op**。padding は導入しない。現状コードに行選択モードも padding も無いことを実コードで確認済み（継承すべき既存挙動なし）。
- **進入方式**: zi に Shift 修飾が無いため、`select-all` と同様の **多段プレフィックス方式**で進入する（方式として確定）。

### 残された曖昧さ（振る舞いレベル）

- なし（この能力の振る舞いは確定）。

### design へ送る実装論点（振る舞いではない）

- 進入の**具体キー割り当て**（多段プレフィックスの第一打鍵）。方式は確定済み、キーは openspec design で確定。
- 2次元アンカーの**状態の持ち方**（既存 `Cursor.selection: Option<CharIndex>` 1個では表現不可、rectangle-mode.md）。「アンカーが (行, visual column) を保持する」ことは振る舞い要請として確定、データ構造は design。
- 矩形モード中の移動キーの範囲: **最低限 上下左右**（対角更新に使える）。行頭/行末・ページ移動の対応有無は実装側裁量。

### 次にどうする

- highlight / copy / cut と合わせて4枚を requirement-definition へ渡せる（振る舞いゲートは閉じた）。
